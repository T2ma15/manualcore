===== SHEET: Schema Overview =====
Row 1 | A: MANUALCORE — DATABASE SCHEMA v1.0
Row 3 | A: SOURCE KEY | B: B = Brain writes (auto)   E = Engineer confirms   S = System generates   C = Calculated   N = Nullable
Row 5 | A: CORE ENTITY TABLES
Row 6 | A: tenants | B: Multi-tenant root. One row per customer account. Holds subscription tier, industry profile, branding.
Row 7 | A: users | B: All users within a tenant. Roles: platform_owner, process_owner, approver, qa, operator, viewer.
Row 8 | A: sessions | B: One session = one process documentation project. Tracks node, industry profile, input files, status.
Row 9 | A: nodes | B: One node = one process or operation. Belongs to a session. Has type, parent node, and link type to next node.
Row 10 | A: MATRIX 1 — MASTER DATA
Row 11 | A: operations | B: One row per operation within a node. Holds op_id, sequence, description, takt inputs, allowance, trained_operator_required.
Row 12 | A: steps | B: One row per step within an operation. Description, material, critical parameter, quality gate flag, safety flag.
Row 13 | A: materials | B: All materials used across steps. Name, SKU, qty_per_op, repetitions, total_qty (computed), unit, notes.
Row 14 | A: product_specs | B: Characteristic, nominal, USL, LSL, unit, measurement method. Linked to steps via quality gate.
Row 15 | A: process_specs | B: Process parameter, value, limits, control method. Linked to operations.
Row 16 | A: specs_5m | B: Man, Machine, Material, Method, Measurement fields. One row per operation.
Row 17 | A: photos | B: Photo or drawing linked to a step. Classification (TWI, WI, safety, drawing), caption, annotation JSON.
Row 18 | A: time_study_observations | B: Raw stopwatch readings. N observations per step. Analyst, rating factor, allowance, derived standard time.
Row 19 | A: MATRIX 2 — DOCUMENT RELATIONSHIPS
Row 20 | A: documents | B: Every generated document. Type, title, doc_number, revision, status, effective_date, review_date.
Row 21 | A: document_relationships | B: Source doc → related doc. Relationship type, shared fields, criticality, last impact check.
Row 22 | A: related_documents_display | B: Denormalised view used by WI and SOP related-docs section. One row per doc-to-doc link for display.
Row 23 | A: MATRIX 3 — OWNERSHIP & APPROVAL
Row 24 | A: ownership | B: Process owner, approver, QA per document. Signature dates. Trainable positions. Training due date.
Row 25 | A: reviewers | B: Many reviewers per document. Name, initials, review date, comment summary.
Row 26 | A: training_records | B: Operator × operation certification. Status C/T/X/N. Certified date. TWI reference. Plateau flag.
Row 27 | A: MATRIX 4 — REVISION & AUDIT
Row 28 | A: revisions | B: One row per revision event. Reason, what changed (brain-generated summary), created_by, approved_by.
Row 29 | A: print_events | B: Every print action. User, copy number, total copies, timestamp, document revision at time of print.
Row 30 | A: access_grants | B: Who has access to which document, at what level (view / print / edit).
Row 31 | A: audit_log | B: Immutable append-only log. Every field write, conflict resolution, status change, and brain action.
Row 32 | A: BRAIN & SESSION TABLES
Row 33 | A: input_files | B: Every file uploaded to a session. Type, language, transcript/OCR result, confidence score.
Row 34 | A: brain_extractions | B: Every field the brain extracted. Source file, extracted value, confidence, status (confirmed/pending/conflict).
Row 35 | A: conflicts | B: Conflict between two extractions. Field name, value_a, source_a, value_b, source_b, resolution, resolved_by.
Row 36 | A: chat_messages | B: Every message in the brain chat. Role (brain/engineer), content, linked conflict or extraction, timestamp.
Row 37 | A: generation_jobs | B: Document generation request. Status, requested_docs, generated_at, output_file_path.
Row 38 | A: DIAGRAM & OUTPUT TABLES
Row 39 | A: diagram_nodes | B: One row per shape in a flowchart or material flowchart. Type, label, position, linked step.
Row 40 | A: diagram_edges | B: Arrow between two diagram nodes. Type (sequential/material/decision-yes/decision-no), label.
Row 41 | A: gantt_tasks | B: One row per Gantt task. Node, task name, type, owner, start_day, end_day, dependency.
Row 42 | A: vsm_data_blocks | B: One row per VSM process box. Cycle time, changeover, uptime, operators, WIP, link type.
Row 43 | A: output_files | B: Generated file records. Format (docx/xlsx/html), path, generated_at, revision snapshot.
Row 45 | A: Total tables: 29   |   Schema version 1.0   |   ManualCore MVP — Manufacturing profile
